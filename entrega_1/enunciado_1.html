def identificar_columnas(df):
    numericas = df.select_dtypes(include=['number']).columns.tolist()
    categoricas = df.select_dtypes(include=['object', 'category']).columns.tolist()
    todas = set(df.columns)
    otras = list(todas - set(numericas) - set(categoricas))
    return numericas, categoricas, otras

En esta seccin, se deben construir pipelines para automatizar el preprocesamiento de los datos, lo cual garantizar que el flujo de trabajo sea reproducible y eficiente para esta entrega y las futuras. El objetivo es aplicar una serie de transformaciones en un orden definido para asegurar que los datos estn listos para los modelos a entrenar.   • Transformaciones personalizadas: Uso de ColumnTransformer para aplicar diferentes transformaciones a columnas especficas.

Cada una de estas transformaciones debe ser justificada en funcin de su relevancia para el problema y los datos, y es importante evaluar su impacto en el rendimiento del modelo. Adems, el pipeline debe ser flexible y modular para poder probar diferentes configuraciones de preprocesamiento.

Implementamos un transformador personalizado (o funcin DateFeatureExtractor) que descompone la variable temporal purchase_date en nuevas caractersticas derivadas: da de la semana, mes y nmero de semana del ao. Esta transformacin busca capturar patrones temporales relevantes que puedan influir en el comportamiento de compra, como estacionalidad o efectos de fin de semana (estacionalidad es el efecto casi peridico de aumento y disminucin de alguna cantidad debido a vacaciones, fechas especiales, etc). Para las variables numricas utilizamos imputacin por mediana seguida de estandarizacin mediante StandardScaler, lo que asegura que las variables estn en una misma escala sin ser afectadas por valores atpicos. En el caso de las variables categricas, imputamos con la moda (por simplicidad y porque estamos recin en el Baseline, y porque luego nos fijaremos en la optimizacin) y luego se codificaron mediante OneHotEncoder, permitiendo convertirlas en una representacin numrica adecuada para los algoritmos de aprendizaje supervisado (que en nuestro caso sern Dummy Classifier, Decision Tree, etc).

from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.impute import SimpleImputer
from sklearn.preprocessing import OneHotEncoder, StandardScaler
from sklearn.base import BaseEstimator, TransformerMixin
import pandas as pd

# ----------------------------
# 1. Extraer caractersticas temporales de la fecha
# ----------------------------
class DateFeatureExtractor(BaseEstimator, TransformerMixin):
    def __init__(self, date_column="purchase_date"):
        self.date_column = date_column

    def fit(self, X, y=None):
        return self

    def transform(self, X):
        X = X.copy()
        X[self.date_column] = pd.to_datetime(X[self.date_column])
        X["dayofweek"] = X[self.date_column].dt.dayofweek
        X["month"] = X[self.date_column].dt.month
        X["week"] = X[self.date_column].dt.isocalendar().week.astype(int)
        return X.drop(columns=[self.date_column])

# ----------------------------
# 2. Definir columnas numricas y categricas finales
# ----------------------------
# Evitar columnas que no deberan ir
excluded_cols = ['target', 'items', 'purchase_date']
numerical_cols = [col for col in numerical_cols if col not in excluded_cols]
categorical_cols = [col for col in categorical_cols if col not in excluded_cols]

# ----------------------------
# 3. Pipelines para cada tipo de dato
# ----------------------------
numeric_transformer = Pipeline(steps=[
    ("imputer", SimpleImputer(strategy="median")),
    ("scaler", StandardScaler())
])

categorical_transformer = Pipeline(steps=[
    ("imputer", SimpleImputer(strategy="most_frequent")),
    ("encoder", OneHotEncoder(handle_unknown="ignore", sparse_output=False))
])

# ----------------------------
# 4. Preprocesador final
# ----------------------------
preprocessor = ColumnTransformer(transformers=[
    ("num", numeric_transformer, numerical_cols),
    ("cat", categorical_transformer, categorical_cols)
])

Queremos un hiperparmetro que maximice la precisin y el recall de la clase minoritaria, y como el F1-score depende de ambas mtricas (precision y recall), entonces nos conviene usar ms el f1-score. Recordemos que la precision es la metrica que habla de la precisin de las prediccione positivas, mientras que el recall habla de la capacidad de identificar todas las intancias positivas. Y es en el recall que queremos comprender la clasificacin correcta de las instancias positivas de la clase minoritaria, pero sin desmedro de clasificar bien la clase 1 (que es la mayoritaria). Por otro lado, hay varios parmetros que la lnea "score = f1_score(y_valid, y_pred, average="weighted")" acepta: entre binary, micro, macro, weighted, samples y None. Nuestro problema est desbalanceado y queremos un F1-score que tenga en cuenta tanto el rendimiento en las clases mayoritarias como en las minoritarias, por lo que la opcin ms adecuada es average='weighted'. A continuacin un resumen matemtico de dicho parmetro weighted: El F1-score ponderado (con average='weighted') se calcula tomando el F1-score de cada clase y luego ponderndolo por el nmero de muestras en cada clase. El F1-score ponderado es el promedio de los F1-score de cada clase, ponderado por el nmero de muestras en cada clase. Esto le da ms peso a las clases con ms ejemplos en el clculo final del F1-score.

    $$ F1_{\text{weighted}} = \frac{1}{N} \sum_{i=1}^{C} N_i \times F1_i $$

    $$ F1_i = 2 \times \frac{Precision_i \times Recall_i}{Precision_i + Recall_i} $$

    $$ Precision_i = \frac{TP_i}{TP_i + FP_i} $$

    $$ Recall_i = \frac{TP_i}{TP_i + FN_i} $$

    Donde:

    • $ N $ es el nmero total de muestras en el conjunto de validacin o prueba.
    • $ C $ es el nmero de clases (en clasificacin multiclase, que en nuestro caso es 2).
    • $ N_i $ es el nmero de muestras de la clase $ i $.
    • $ F1_i $ es el F1-score calculado para la clase $ i $.
    • $ TP_i $ = Verdaderos positivos para la clase $ i $
    • $ FP_i $ = Falsos positivos para la clase $ i $
    • $ FN_i $ = Falsos negativos para la clase $ i $
